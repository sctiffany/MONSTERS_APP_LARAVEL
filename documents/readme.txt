Cahier des charges : Application RetroMonsters

1. Présentation du projet
RetroMonsters est une application web permettant de gérer une liste de monstres imaginaires, avec des informations telles que leur nom, leur type (aquatique, terrestre, cosmique, etc.), leurs statistiques (PV, attaque, défense), et une brève description.

L’objectif est de réaliser un mini-projet en Laravel, incluant les fonctionnalités CRUD de base, en respectant un design HTML/CSS fourni.

2. Contexte et contraintes

Technologies imposées :
Laravel (framework PHP côté back-end).
MySQL pour la base de données.

Hébergement :
Le projet doit être en ligne, hébergé sur un serveur accessible (Heroku, OVH, ou tout autre hébergement web).

Front-end :
Utiliser et respecter le template HTML/CSS fourni (basé sur Tailwind CSS, etc.).

3. Fonctionnalités obligatoires
3.1. Page d’accueil dynamique
Affichage d’une liste de monstres dynamique (derniers monstres ajoutés, monstre aléatoire).

3.2. La page des monstres est la liste dynamique des 9 derniers monstres.

3.3. Détails d’un monstre
Lorsqu’on clique sur un monstre, on accède à sa page de détails.
Cette page doit afficher toutes les informations du monstre : nom, type, PV, attaque, défense, description…

3.3. Ajout d’un monstre
Un formulaire permettant de saisir un nouveau monstre dans la base de données.
Valider a minima le nom (obligatoire) et le type (obligatoire).
Image en bonus.

3.4. Modification d’un monstre
Une page/formulaire permettant de modifier les informations d’un monstre existant (même format que l’ajout).
Mettre à jour la base de données.
Image en bonus.

3.5. Suppression d’un monstre
Possibilité de supprimer un monstre de la base de données.
Important : Il n’y a pas de système de connexion / d’utilisateurs. Tout le monde peut gérer les monstres.

4. Fonctionnalités optionnelles (non obligatoires, bonus)
Recherche et Filtres
Pagination de la page "monstres"
Upload de l'image lors de l'ajout et la modification

5. Fonctionnalités à ne pas développer
Commentaires : pas de section commentaires ou d’ajout de commentaires.
Ajouter au deck : on ne gère pas la notion de deck ni d’utilisateur.
Évaluation / notation par étoiles.
